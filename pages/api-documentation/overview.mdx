## Response Data Structure

The response data follows the following structure throughout the API:

### Success Response

```typescript
{
    "status": 200;
    "data": object;
}
```

### Error Response

```typescript
{
    "status": 500| 400 | 401;
    "message": string;
    "data": object | undefined;
}
```

---

## API Endpoints

The API currently supports the followig endpoints:

### Utility

-   <Badge>`GET:/api`</Badge> - Get information about the API and list its endpoints
-   <Badge>`GET:/api/ping`</Badge> - Check if the server is running

### Data

-   <Badge>`GET:/api/schedule/:id`</Badge> - Get personalised schedule for a specific user
